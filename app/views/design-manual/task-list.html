Task list

A task list breaks up long journeys into manageable sections and shows users what information is needed at each stage.

GOV.UK Design System

Task lists appear in the GOV.UK Design System as the complete multiple tasks pattern.

When to use it

This pattern is used in longer or more complex journeys where users need to complete several steps to provide all required information.

A task list helps users:

  • understand what information is needed
  • return to previously completed sections to review or change answers
  • complete the journey in stages

How it works

The task list is shown at the start of a journey, after the user selects a warrant type. Each task represents a section of the journey. Users can return to it to view or edit their answers without navigating back step by step.

Defining tasks

Tasks should be specific activities in the overall journey, such as adding offences or entering warrant information. Task names should clearly describe the activity and ideally start with a verb, such as "Add offences".

Do not group multiple activities into a single task. It is easier for users to complete shorter, specific tasks than longer tasks with multiple unrelated actions.

Page title and instructions

The page title for a task list should be the name of the journey.

Provide brief, clear instructions to help the user understand what they are required to do on this journey and the overall goal the tasks relate to.

Task status

  • Tasks that are required, and do not depend on answers from other tasks, start with an "Incomplete" status.
  • Tasks that are optional are shown with an "Optional" status.
  • Tasks that require information from other tasks that have not yet been completed have a status of "Cannot start yet", and the task name is shown as plain text.
  • When a task is completed, it is shown with a "Completed" status.
