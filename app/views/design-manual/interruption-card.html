Interruption card MoJ Design System

Use the interruption card to pause the user’s journey and draw attention to something important about the action they are performing.

When to use it

Use this component in Remand and Sentencing to help prevent mistakes when a user’s action could affect other parts of the service, such as breaking a consecutive sentence chain.

Only use an interruption card when the user has a choice. They should always have the option to cancel the action and go back a step.

Ministry of Justice Design System

The interruption card is part of the Ministry of Justice Design System. Read more on the MoJ Design System website.

Use sparingly

Interruption cards should only be used when there’s no better way to prevent errors. Service design should minimise the chance of user actions leading to problems elsewhere.

Make sure there is a clear use case and avoid using them too often, as users may start to ignore them (known as banner blindness).

When not to use it

Do not use an interruption card to block users from performing an action. It should highlight risks, not prevent action.

How it works

The title of the interruption card should briefly describe the potential issue. If the journey is in the context of something specific (like a sentence), include this information—similar to the contextual question pattern.

Give clear instructions that explain the potential impact elsewhere in the service and what the user may need to do as a result.

The card should ask whether the user wants to continue with the action. Provide a clear primary call to action and always offer a way to go back or cancel.
